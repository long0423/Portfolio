SUMMARY

I created the web application prototype and lead UI design and a part of the front-end coding

Problem Statement

What our client wanted is a personally customized web application that meets the client's daily work schedule. In view of the already existing scheduler applications on the market, our client hopes that this customized application simplifies the operation steps for end users and removes useless and cumbersome functions as in others with a flexibility for future function expansion.

Given that most end users of our client are middle-aged and elderly workers, both of our dev team and design team had to be careful to make all functionalities and procedures simplified and intuitional because some of them don't even know how to use a smartphone which brings a real challange when it comes to system design.

Project Goals

Simplify application logic & reduce operation steps

Build connection between the host end and the mobile end

Bring "drag and drop" functionality to job scheduling

Establish trust and usability through excellent UI

Capacity and flexibility for future functionality expansion

INTERACTION OPTIMIZATION

We optimized all the modals for job creation and assignment and improved the drag and drop interaction after several attempts to help aid the end-users to get a smooth experience.

Also we try to make the link page as similar as app experience when users click the link in text messages to accept or decline the job that has been assigned to them..

Defining the MVP

Through close collaboration with the lead developer, I was able to fine tune the complexity of the navigation and using Principle App I was able to accurately communicate subtle but necessary interactions.

Atomic consistency

Alongside the finished interface designs, I produced a sketch file documenting all elements to keep the designs as consistent and re-usable as possible. This was an important tool for the development team to use and build an understanding for.

Learnings

Seek critical feedback early

Despite the close collaboration between design and build, there could have been more honest conversations had early on when defining the MVP. Our ambition blinded us early, which resulted in inadequate time for build feedback and a premature release.
