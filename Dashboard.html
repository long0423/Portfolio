SUMMARY

I re-designed the UI of the integrated dashboard system and front end coded with React JS in Material UI

Problem Statement

IMS is the dashboard system used by our client to manage their road construction files. The client already has a basic version of the PHP-based system which was built long time ago. Both user interface and operation experience are awful, so we plan to re-code using the popular React framework combined with the user-friendly Material UI to upgrade the entire system.

This upgrade not only brings a clearer and more efficient UI, but also improves system functionality and user security. We also spend a lot of time and energy fixing all the left over bugs for future function expansion.

Mobile Optimization

The benefit of using React together with React-bootstrap is that it's friendly with mobile device layout. The old version has a messed up responsive design with poor user experience. Now we are bringing the responsive layout to a new level.

Atomic Consistency

Thanks to the convenience brought by Material UI Library, we can use all the components in this library by importing the API, so that the entire theme can be consistent and re-usable for future feature expansion.

Learnings

Plan first then code

It’s easy to kickstart a simple and working React prototype with a knowledge base of HTML, CSS and JavaScript, but can get messy once you start having more states to track. An architectural plan is required if you don’t want to code yourself into a corner. In a way, that is how things are supposed to be done — plan first then code — but that’s not the case in a lot of projects, so it's a good React practice and will keep digging into it.
